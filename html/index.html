MIT

API to route WPILib HAL calls over WebSockets.

Operations

Pub wpilibws

General channel for WPILib WebSocket messages

Accepts the following message:

WPILib WebSocket Message

Message envelope. Note that the "data" field contains a diff of the current state of a particular device. E.g. If only the "value" changes for a DIO device, then only the "<>value" field will be sent.

Payload
object
Examples: { "type": "PWM", "device": "1", "data": { "<speed": 0.5 } } { "type": "DIO", "device": "3", "data": { "<init": true } }
type
required
string

Device Type (e.g. DIO/AO/AI/PWM/Encoder etc)

device
required
string

Device Identifier (usually channel)

data
oneOf

Payload for the message. Note that not all fields for a particular message will get sent. Only the diff of the current state of the particular device will be sent.
